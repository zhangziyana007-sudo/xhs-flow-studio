# FlowStudio — 小红书图文创作工作台

> 版本：v0.3 (2025-05-31)
> 前身项目：AINewsSkill v3.1.0

---

## 1. 产品概述

### 1.1 定位

FlowStudio 是一个**专注小红书图文创作的桌面工作台**。创作者打开工作台，从"找素材"到"出图发布"的整条链路都在这里完成。

它不是一个通用自动化工具，而是**创作者每天打开、每天用的生产力工具**。

### 1.2 核心模型

**工作台的根基是一条明确的流水线**：

```
素材输入  ──→  AI 自由创作  ──→  图文输出
(确定的)       (开放的)         (确定的)
```

两端是**确定的工程能力**（怎么获取素材、怎么渲染出图），中间是**开放的创作空间**（AI + 用户协作，不设限）。

这就像给创作者一个画室：
- 画架和颜料（素材获取 + 渲染引擎）是固定的工具
- 怎么画（AI 创作过程）完全自由

### 1.3 核心场景

一个典型的创作日：

```
打开工作台
  → 看看今天有什么热点/灵感（素材面板）
  → 跟 AI 说"帮我做成小红书图文"
  → AI 自由发挥，给出内容方案
  → 觉得不对就聊几句调整
  → 选模板、预览效果
  → 满意了，导出图片 + 文案
  → 定时的任务也在后台自动跑着
```

### 1.4 设计原则

| 原则 | 含义 |
|------|------|
| 管道确定，创作自由 | 两端（输入/输出）是可靠的工程，中间是不设限的 AI 创作空间 |
| 卡片化 UI | 所有信息以卡片形式呈现，可扫视、可操作、可拖拽 |
| 所见即所得 | 模板预览实时可见，调参即刻更新 |
| 低门槛 | 不需要懂代码，打开就能用 |
| 可自动化 | 重复性创作任务可设定时自动执行 |

### 1.5 UI 风格 — 卡片优先

整个工作台的 UI 统一采用**卡片（Card）**作为信息承载单元：

```
┌────────────────────────────────────┐
│  ┌──────┐  ┌──────┐  ┌──────┐    │ ← 素材卡片网格
│  │素材1  │  │素材2  │  │素材3  │    │
│  │      │  │      │  │      │    │
│  └──────┘  └──────┘  └──────┘    │
│  ┌──────┐  ┌──────┐              │
│  │素材4  │  │素材5  │              │
│  └──────┘  └──────┘              │
└────────────────────────────────────┘
```

| 面板 | 卡片内容 |
|------|----------|
| 素材面板 | 每条素材 = 一张卡片（标题 + 摘要 + 来源标签 + 时间） |
| 加工面板 | AI 对话气泡卡片 + 快捷模式选择卡片 |
| 样式面板 | 每个模板 = 一张预览卡片（缩略图 + 名称 + 标签） |
| 预览面板 | 每张图 = 一张大卡片（可翻页浏览）+ 底部"保存为任务" |
| 任务面板 | 每个任务 = 一张状态卡片（名称 + 触发方式 + 运行状态 + 操作） |

**卡片设计规范**：
- 圆角：24px（`rounded-3xl`），按钮 16px（`rounded-2xl`）
- 阴影：三层叠加立体阴影（近距锐影 + 中距扩散 + 远距柔化）
- hover 效果：阴影加深 + 上浮 6~8px（`-translate-y-1.5`）
- 间距：卡片间 16px
- 按钮：品牌色投影（蓝色按钮带蓝色发光阴影，绿色同理）
- 空状态：96px 渐变容器 + 内阴影（磨砂玻璃质感）
- 交互：点击展开详情 / 长按多选 / 右键菜单
- 响应式：根据窗口宽度自适应列数（2~4列）
- 状态：选中态蓝色描边、禁用态灰度

## 2. 工作台功能设计

工作台的核心逻辑：**创作流程（4步）= 任务编辑器 + 调试器**，完成后保存为任务。

### 2.0 整体布局与导航

```
┌─────────────────────────────────────────────────────────────┐
│                    FlowStudio 工作台                          │
├───────┬─────────────────────────────────────────────────────┤
│       │  [素材] → [AI创作] → [样式] → [出图]  ← 顶部进度条  │
│ 侧    ├─────────────────────────────────────────────────────┤
│ 边    │                                                     │
│ 导    │         当前面板内容区                                │
│ 航    │                                                     │
│       │                                                     │
│ ┌───┐ │                                                     │
│ │创作│ │                                                     │
│ │ 1│ │                                                     │
│ │ 2│ │                                                     │
│ │ 3│ │                                                     │
│ │ 4│ │                                                     │
│ └───┘ │                                                     │
│ ┌───┐ │                                                     │
│ │任务│ │                                                     │
│ └───┘ │                                                     │
└───────┴─────────────────────────────────────────────────────┘
```

**导航结构**：

| 层级 | 内容 | 说明 |
|------|------|------|
| 一级 | 创作 | 可展开/折叠，包含 4 步子功能 |
| 二级 | 素材 / AI创作 / 样式 / 预览出图 | 创作的 4 个步骤 |
| 一级 | 任务 | 管理已保存的任务列表 |

**核心工作流**：
```
用户在「创作」中走完 4 步
  → 配置好素材源、AI创作参数、样式模板
  → 在预览面板确认产出效果
  → 点击"保存为任务"
  → 任务进入「任务」列表
  → 可设为定时自动运行 / 手动一键运行
```

**侧边栏视觉**：
- 背景：浅灰（`bg-gray-50`），让激活态白色卡片浮出
- 一级标签：激活时白底 + 品牌色投影 + 蓝色图标方块发光
- 二级步骤：左侧竖线连接 + 编号方块 + 激活态蓝色底
- 已完成步骤：绿色对勾 + 绿色方块
- 毛玻璃效果：`backdrop-blur-sm`

### 2.1 素材面板 — "今天做什么内容？"

创作者在这里获取和管理原始素材。

**素材获取方式**：

| 方式 | 操作 | 适用场景 |
|------|------|----------|
| 热点抓取 | 点击"获取今日热点"，自动从 AI HOT / 各平台获取 | 日报、热点解读 |
| RSS 订阅 | 配置 Feed 列表，工作台自动拉取更新 | 行业资讯追踪 |
| 粘贴输入 | 直接粘贴文字/Markdown 到输入框 | 读书笔记、自己写的内容 |
| 文件拖入 | 拖入 .md / .txt / .json 文件 | 已有文档转图文 |
| 网页采集 | 输入 URL，自动提取正文 | 将文章转为图文 |
| 截图识别 | 粘贴/拖入截图，OCR 提取文字 | 快速录入灵感 |

**素材管理**：
- 素材以卡片形式展示，可标记星标、添加标签
- 按日期/来源/标签筛选
- 素材库持久保存，随时回溯

### 2.2 加工面板 — "AI 帮我创作"

AI 在这里不是单纯的格式化工具，而是**有创作自主权的合作者**。

**设计理念**：用户给方向，AI 决定怎么做。

**交互方式：对话式创作**

```
┌─────────────────────────────────────────┐
│ 素材已导入（3条AI新闻 + 1段我的笔记）     │
├─────────────────────────────────────────┤
│                                         │
│  用户：帮我做成小红书图文，要有科技感      │
│                                         │
│  AI：我看了你的素材，建议这样组织：        │
│      - 封面用一句话概括今天最大的事        │
│      - 中间3页按影响力排序讲3个新闻        │
│      - 最后一页加你的个人点评              │
│      总共5张图，要不要我开始？              │
│                                         │
│  用户：好的，语气毒舌一点                  │
│                                         │
│  AI：[生成结构化内容...]                  │
│                                         │
└─────────────────────────────────────────┘
```

**AI 的自由度**：

| AI 可以自主决定的 | 说明 |
|-------------------|------|
| 内容结构 | 不限于预设模式，AI 根据素材自行判断最佳呈现方式 |
| 分页策略 | AI 决定做几张图、每张图放什么内容 |
| 标题文案 | AI 自由创作标题和文案风格 |
| 信息取舍 | AI 判断哪些素材值得用、怎么组合 |
| 叙事角度 | AI 可以选择不同的切入点和叙事方式 |

**用户的控制权**：

| 用户可以控制的 | 方式 |
|----------------|------|
| 大方向 | 自然语言描述（"做成科技日报""写得可爱一点"） |
| 约束条件 | "最多5张图""要包含某条新闻""不要太长" |
| 迭代修改 | "第二页重写""换个角度""加点emoji" |
| 模板建议 | AI 生成后会推荐适合的模板，用户可改 |

**预设模式（可选捷径，不是强制）**：

用户赶时间时可以直接选预设模式，跳过对话：

| 快捷模式 | AI 会怎么做 |
|----------|-------------|
| 📰 日报速出 | 按时间排列，摘要式，5~8张图 |
| 💡 知识提炼 | 提取核心观点，卡片式，3~5张图 |
| 🛒 种草安利 | 突出卖点优缺点，对比式，4~6张图 |
| 📚 读书精华 | 金句+感悟，文艺式，3~5张图 |
| 🔥 热点快评 | 事件+观点，冲击力式，3~4张图 |
| ✏️ 完全自由 | AI 全权决定，用户只给素材 |

**LLM 配置**（设置里配置，创作时不操心）：
- 默认 LLM：DeepSeek（可切换 OpenAI / Claude）
- API Key 在设置面板统一管理
- 支持自定义 API 端点（本地模型/代理）

### 2.6 设置面板 — AI 模型配置

创作者在这里配置自己的 LLM API，工作台通过 Vercel AI SDK 统一调用。

Vercel AI SDK 的优势：
- 内置多 Provider 支持（OpenAI / Anthropic / Google / Mistral / 自定义）
- 统一的流式输出接口（创作过程实时显示 AI 输出）
- 内置结构化输出（`generateObject`），确保生成的内容符合模板数据结构
- 支持工具调用（AI 可以主动搜索素材、查询信息）
- TypeScript 原生支持，类型安全

**模型配置界面**：

```
┌─────────────────────────────────────────────────┐
│  AI 模型管理                          [+ 新增]   │
├─────────────────────────────────────────────────┤
│                                                 │
│  ⭐ DeepSeek（默认）                    [编辑]   │
│     端点：https://api.deepseek.com/v1           │
│     模型：deepseek-chat                         │
│     状态：✅ 已连接                              │
│                                                 │
│  ── OpenAI                              [编辑]   │
│     端点：https://api.openai.com/v1             │
│     模型：gpt-4o                                │
│     状态：✅ 已连接                              │
│                                                 │
│  ── 本地 Ollama                         [编辑]   │
│     端点：http://localhost:11434/v1             │
│     模型：qwen2.5:32b                           │
│     状态：⚠️ 未连接                              │
│                                                 │
│  ── 自定义（中转站）                    [编辑]   │

│     状态：✅ 已连接                              │
│                                                 │
└─────────────────────────────────────────────────┘
```

**每个模型的配置项**：

| 配置项 | 说明 |
|--------|------|
| 名称 | 显示名（如"DeepSeek""公司中转"） |
| API 端点 | OpenAI 兼容的 base URL |
| API Key | 密钥（本地加密存储） |
| 模型标识 | model 参数（如 `deepseek-chat`、`gpt-4o`） |
| 最大 Token | 输出 token 上限 |
| 温度 | temperature 默认值（0~2） |
| 设为默认 | 是否作为创作时的默认模型 |

**支持的接入方式**（Vercel AI SDK 内置 Provider）：

| 方式 | Provider | 说明 |
|------|----------|------|
| DeepSeek | `@ai-sdk/openai` 兼容模式 | 默认推荐，性价比高 |
| OpenAI | `@ai-sdk/openai` | GPT-4o 等 |
| Claude | `@ai-sdk/anthropic` | 原生支持，无需中转 |
| Google Gemini | `@ai-sdk/google` | Gemini Pro 等 |
| 本地模型 | `@ai-sdk/openai` 兼容模式 | Ollama / LM Studio / vLLM |
| 自定义端点 | `createOpenAI({ baseURL })` | 任何 OpenAI 兼容 API |
| 聚合平台 | `createOpenAI({ baseURL })` | OpenRouter / one-api |

**连接测试**：
- 配置完成后一键"测试连接"，发送简单请求验证可用性
- 显示响应时间和 token 计费信息
- 连接失败给出明确错误提示

**安全**：
- API Key 使用系统 Keychain 加密存储（Electron safeStorage）
- 界面上 Key 默认遮蔽显示（`sk-****xxxx`）
- 配置文件不明文存储密钥

### 2.4.1 发布功能设计

任务运行后的产出物需要通过发布渠道送达目标平台。发布是任务管道的最后一环。

**三种发布模式**：

| 模式 | 场景 | 行为 |
|------|------|------|
| 立即发布 | 定时任务无人值守 | 运行完自动推送到配置的渠道 |
| 待审核 | 想看看再发 | 运行完先存队列，用户确认后才发布 |
| 仅保存 | 只做不发 | 只输出到本地目录，不推送任何渠道 |

**发布渠道**：

| 渠道 | 实现方式 | 产出物 |
|------|----------|--------|
| 本地保存 | 直接写文件 | PNG 图片组到指定目录 |
| 小红书发布 | 打包素材文件夹 | 图片 + 标题 + 正文 + 话题标签 .txt |
| 飞书群 | Incoming Webhook | 图片 + 文案消息卡片 |
| 企微群 | Webhook | 图片 + Markdown 消息 |
| 钉钉群 | Webhook | 图片 + 富文本消息 |
| API 更新 | REST PUT/POST | 更新远程数据接口 |
| 自定义 Webhook | HTTP POST | 用户自定义 payload 模板 |

每个任务可配置**多个渠道同时发布**，每个渠道独立开关。

**发布配置数据结构**：

```typescript
interface PublishChannel {
  id: string
  type: 'local' | 'xhs-package' | 'feishu' | 'wecom' | 'dingtalk' | 'api' | 'webhook'
  name: string              // 显示名："飞书-AI日报群"
  enabled: boolean
  config: {
    webhookUrl?: string     // Webhook 地址
    apiEndpoint?: string    // API 端点
    directory?: string      // 本地目录
    includeText?: boolean   // 是否附带文案
    template?: string       // 自定义消息模板
  }
}

// 在 OutputConfig 中：
interface OutputConfig {
  format: 'png' | 'pdf' | 'html'
  exportPath?: string
  publishMode: 'immediate' | 'review' | 'save-only'
  channels: PublishChannel[]
}
```

**待审核队列**（任务面板内展示）：

当有任务产出物等待审核时，任务面板顶部显示审核区域：

```
┌──────────────────────────────────────────────────────────┐
│  待发布 (2)                                               │
├──────────────────────────────────────────────────────────┤
│  ┌────────────────────────────────────────────────────┐  │
│  │ 每日AI热点日报 · 今天 08:01 产出                     │  │
│  │ [缩略图] [缩略图] [缩略图] ...共5张                  │  │
│  │                                                    │  │
│  │ [预览]  [确认发布]  [编辑后发]  [丢弃]              │  │
│  └────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────┘
```

**发布流程**：
```
任务运行完成
  → publishMode === 'immediate' → 直接推送所有 enabled 渠道
  → publishMode === 'review'    → 存入待审核队列 → 用户确认 → 推送
  → publishMode === 'save-only' → 仅保存到本地 exportPath
```

### 2.5 任务面板 — "管理我的创作任务"

任务面板展示用户保存的所有任务。**创作流程 = 任务编辑器**，走完后保存到这里。

> 设计原则：定时任务是确定性行为，用户明确指定"什么时候做什么"，系统严格按规则执行。AI 只在任务执行中的"创作"环节发挥作用，不能自主决定何时执行、发布到哪里、是否跳过。

**任务数据结构**：
```typescript
interface Task {
  id: string
  name: string
  description?: string
  createdAt: number
  updatedAt: number
  pipeline: {
    source: { type, keywords?, rssUrl?, filePath?, text? }
    generate: { model, mode, prompt, temperature? }
    style: { templateId, overrides? }
    output: { format, exportPath?, publish? }
  }
  trigger: {
    type: 'scheduled' | 'manual'
    cron?: string       // "0 8 * * *" = 每天早8点
    enabled: boolean    // 定时开关
  }
  lastRun?: { at, status, outputPath?, error? }
  runCount: number
}
```

**任务来源**：
- 在预览面板完成创作后，点击"保存为任务" → 自动填充管道配置
- 在任务面板点击"新建任务" → 从头配置

**触发方式**：

| 类型 | 行为 | 控制 |
|------|------|------|
| 定时（scheduled） | `node-cron` 按 cron 表达式自动执行 | 可暂停/启用开关 |
| 手动（manual） | 用户点击"运行"一键执行 | 配置已存好，无需重新设置 |

**任务卡片 UI**：
```
┌────────────────────────────────────────────────────────┐
│  每日 AI 热点日报          [定时] ●                     │
│  自动抓取AI领域热点，生成科技风日报图文                  │
│                                                        │
│  ⏰ 0 8 * * *  ·  上次运行: 1小时前  ·  已运行 12 次    │
│                                                        │
│  [🔘开关]  [▶ 运行]  [✏ 编辑]  [🗑 删除]               │
└────────────────────────────────────────────────────────┘
```

**任务操作**：

| 功能 | 说明 |
|------|------|
| 运行 | 立即执行一次（不等定时） |
| 编辑 | 打开创作流程，加载该任务的管道配置进行修改 |
| 复制 | 基于现有任务创建副本 |
| 删除 | 删除任务 |
| 开关 | 启用/暂停定时触发 |

**任务存储**：
- 本地 JSON 文件：`~/.flowstudio/tasks/{id}.json`
- 轻量无数据库，方便跨平台迁移

**自动化边界（明确规定）**：

| 自动化的 | 不自动化的 |
|----------|------------|
| 按时触发任务 | AI 不能自行决定何时触发 |
| 按配置获取素材 | AI 不能自行更换素材源 |
| 按指令调用 AI 创作 | AI 不能改变创作指令 |
| 按配置选模板渲染 | AI 不能自行换模板 |
| 按配置导出/发布 | AI 不能自行选择发布渠道 |
| 失败时按规则重试 | AI 不能绕过重试限制 |

**典型任务示例**：
- "每天早上8点自动出一期 AI 日报，发到飞书群"
- "每周一出一期行业周刊，保存到本地"
- "每天中午获取热点，存到素材库等我下班编辑"
- "每天18点生成图文但不发布，等我确认后再发"

## 3. 技术架构

### 3.1 技术选型

| 层 | 选型 | 理由 |
|----|------|------|
| 桌面框架 | Electron 31+ | 复用 Node.js 生态，跨平台 |
| 前端 | React 18 + Tailwind CSS 3.4 | 组件化 + 快速 UI |
| 构建 | Vite 5 + electron-vite 2.3 | HMR + 开发体验 |
| 截图引擎 | Playwright | 高保真 HTML→PNG |
| 模板引擎 | Handlebars | 简单够用，非程序员也能改 |
| 定时 | node-cron | 轻量可靠 |
| LLM | Vercel AI SDK (`ai@^4.3`) | 统一多模型接口，内置流式输出/工具调用/结构化输出 |
| 状态 | Zustand 5 | 轻量 |
| 图标 | Lucide React | 统一矢量图标，不使用 emoji |
| 存储 | 本地 JSON 文件（`~/.flowstudio/`） | 无服务器依赖，方便迁移 |
| 包管理 | npm | 通用 |


## 4. 与 AINewsSkill 的关系

| 可复用 | 说明 |
|--------|------|
| 流水线模式 | fetch → generate → render → shot → publish 同样适用 |
| AI HOT 素材源 | 直接迁移为"热点抓取"功能 |
| DeepSeek 调用 | 迁移为 LLM Client 统一层 |
| HTML 模板 + Playwright 截图 | 核心渲染链路完全复用 |
| 飞书 Webhook | 迁移为发布插件 |
| 小红书素材包生成 | 迁移为导出功能 |
| Tech Neon 模板 | 作为首个内置模板 |

**不复用 / 重写**：
- CLI 架构（改为 Electron 桌面应用为主，CLI 为辅）
- 单包结构（改为模块化的 src 结构）
- 硬编码配置（改为 UI 可配置）
